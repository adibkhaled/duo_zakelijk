# Allure Report Integration Guide

## Overview

Allure Report is a flexible, lightweight multi-language test report tool that provides clear graphical representation of what has been tested, and various details of test execution, figures and trends.

## Installation

All Allure dependencies are already installed:
- `allure-playwright` - Allure Playwright adapter
- `allure-commandline` - Allure command line tool
- `rimraf` - For cleaning directories

## Configuration

The Allure reporter is configured in `playwright.config.ts`:

```typescript
reporter: [
  ['html', { outputFolder: 'playwright-report' }],
  ['allure-playwright', { outputFolder: 'allure-results' }],
],
```

## Available Commands

### Run Tests and Generate Allure Results
```bash
npm test
```

### Generate and View Allure Report
```bash
npm run allure:report
```

This command will:
1. Generate the Allure report from test results
2. Open the report in your default browser

### Generate Allure Report (without opening)
```bash
npm run allure:generate
```

### Clean Allure Results and Report
```bash
npm run allure:clean
```

### Run Tests on Specific Browser and View Allure Report
```bash
npm run test:chromium
npm run allure:report
```

or

```bash
npm run test:edge
npm run allure:report
```

## Directory Structure

After running tests, you'll have:

```
allure-results/          # Raw test data (generated by Allure Playwright)
├── *.json              # Individual test result files
├── history/            # Test history for trends
└── ...

allure-report/          # Generated HTML report
├── index.html         # Main report page
├── plugins/           # Report plugins and features
├── css/               # Styling
├── js/                # JavaScript files
└── ...

playwright-report/      # HTML report (traditional Playwright format)
└── index.html
```

### 1. Test Results Overview
- Total tests executed
- Pass/Fail/Skip statistics

- Flaky test detection

### 2. Test Categories and Grouping

### 3. Test Details
- Test steps (if added in code)
- Attachments (screenshots, videos)
- Parameters used
- Logs and error messages
- Duration

### 4. Trends and History
- Test execution trends
- Pass/Fail trends over time
- Flaky test detection
- Duration trends

### 5. Timeline View
- Sequential test execution view
- Parallel execution details
- Duration breakdown

## Adding Test Details to Code

You can enhance Allure reports by adding more information in your test code:

```typescript
import { test, expect } from '@playwright/test';

test('Test with Allure details', async ({ page }) => {
  // Add test step
  await test.step('Navigate to page', async () => {
    await page.goto('https://duo.nl/zakelijk/');
  });

  // Add test step
  await test.step('Verify page loaded', async () => {
    await expect(page.getByRole('link', { name: 'Zakelijk' })).toBeVisible();
  });
});
```

## Troubleshooting

### Allure Command Not Found
```bash
# Use npx to run allure commands
npx allure --version
npx allure generate allure-results -o allure-report --clean
npx allure open allure-report
```

### No Allure Results Generated
- Ensure tests ran successfully: `npm test`
- Check that `allure-results/` folder exists
- Verify playwright.config.ts has Allure reporter configured

### Report Not Opening Automatically
```bash
npm run allure:generate
# Then manually open allure-report/index.html in browser
```

## Continuous Integration (CI)

For CI/CD pipelines:

```bash
# Run tests with Allure reporting

```

## Best Practices

1. **Add Test Steps** - Use `test.step()` to add detailed steps to Allure report
2. **Use Severity Levels** - Tag tests with severity for better organization
3. **Regular Cleanup** - Use `npm run allure:clean` before running fresh test suites
4. **Archive Reports** - Keep historical reports for trend analysis
5. **CI Integration** - Generate Allure reports in CI pipeline for each build

## Integration with Test Code

### Example with Severity Tags
```typescript
import { test, expect } from '@playwright/test';

test('Critical test', async ({ page }) => {
  test.info().annotations.push({ type: 'severity', description: 'critical' });
  
  await test.step('Step 1', async () => {
    // Test code
  });
});
```
